結論摘要

此次改動的核心功能是：新增 AbortAndChangePosition 多軸同步終止並切換位置能力，並將 RA605 的持續移動控制從「PVT 批次軌跡」改為「CSP 速度控制迴圈」。

這代表系統現在可以在持續移動過程中，直接清空既有佇列、同步煞停，並切換到指定關節位置；同時持續移動流程也改為每 100ms 根據虛擬末端姿態與 IK 結果即時計算關節速度，透過 ChangeVelocity 推進。

新功能二：RA605 持續移動模式改為 CSP 速度迴圈

最關鍵的行為調整在 Robot.Motion.RA605/MotionController.cs。原本的持續移動流程會組 PVT 批次點、估算批次起終速度，再用 MoveMultiAxisPVT 送出。現在整段流程被替換成 CSP 型式的即時控制。

  • 新增 100ms 週期常數與每拍最大速度變化量 CSP_MAX_VEL_STEP_MDEG。
  • 進入持續模式時，先讓每軸朝離當前位置較遠的軟體限位做超低速絕對移動，藉此進入 CSP 可接管的狀態。
  • 每個週期先累積「虛擬末端姿態」，再用 IK 算出目標關節角。
  • 用 (target - current) / 0.1s 算出期望關節速度，並對速度變化量做限幅，最後逐軸送 ChangeVelocity。
  • 當增量歸零、IK 無解、或控制迴圈例外時，不再只是停住或截斷 PVT，而是呼叫 AbortAndChangePosition(AxisCard.Pos, 0.3) 做穩定停機。
被移除的舊策略 舊版的 PVT 批次點生成、批次首尾速度銜接、關節跳動檢查與 Ramp() 緩升函式都已退出主流程。
新的控制核心 改成以目前編碼器位置為回授，每拍重新計算速度命令，更接近即時伺服控制。
停止策略改變 停止邏輯從逐軸 Stop() 改為多軸同步 AbortAndChangePosition()。

實際影響

  • 持續移動的控制模型改變，系統行為不再依賴 PVT 軌跡批次平滑銜接，而改由 100ms 速度閉環樣式的更新驅動。
  • 系統新增一個可從上層直接呼叫的「同步終止並改位置」能力，適合用於平順停機、模式切換或持續移動收尾。
  • RA605 的持續移動一旦收到零增量或 IK 失敗，會主動進入穩定停機，不再只是等待下一批次或保留既有佇列。
