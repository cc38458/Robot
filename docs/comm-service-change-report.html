4. 看門狗行為

4.1 主程式結束 → CommService 安全關機

觸發條件：管道斷線 或 主程式心跳逾時（>1s）

SafeShutdown() 程序：
  Step 1: _driver.Estop()      — 全軸緊急停止
  Step 2: _driver.End()        — Servo OFF + 關閉 EtherCAT 連線
  Step 3: _driver.Dispose()    — 釋放所有資源
  → CommService 行程自行終止

4.2 CommService 崩潰 → 主程式警報

偵測方式（三重保護）：

  • 共享記憶體心跳逾時：CommService 心跳 >1s 未更新
  • 行程監控：Process.HasExited 偵測到行程終止
  • 管道讀取失敗：ReadLine() 返回 null

處置措施：

PipeAxisCard 偵測到崩潰後：
  1. 設置 _commServiceDead = true
  2. AxisCardState 回報 ALARM
  3. 所有後續指令返回 false
  4. IsCommServiceDead 屬性供上層查詢

上層（RA605RobotApp）應：
  1. 偵測到 ALARM 狀態或 IsCommServiceDead
  2. 立即進行 in-process 降級（建立 DeltaDriver 直接接管軸卡）
  3. 透過直連軸卡發送 Estop 確保安全
  ※ 重新啟動 CommService 對 EtherCAT 連線無幫助（連線已建立在崩潰的行程中）

5. 各檔案詳細變動

5.1 Robot.Core/IPC/PipeProtocol.cs

命名管道 JSON Line 協定定義。

  • PipeProtocol 靜態類：管道名稱常數、JSON 序列化/反序列化工具
  • PipeRequest：請求物件（Id, Cmd, 各種運動參數），對應 IAxisCard 全部 13 個方法
  • PipeResponse：回應物件（Id, Ok, Error）
  • 管道名稱：RobotCommService
  • JSON 選項：camelCase 命名 + 忽略 null 值（減少傳輸量）

5.2 Robot.Core/IPC/SharedMemoryState.cs

跨行程共享記憶體，使用 file-backed MemoryMappedFile（相容 WSL2/Windows）。

  • SeqLock 模式：寫端將序號 +1（奇數）→ 寫入資料 → 再 +1（偶數）；讀端比較前後序號，不一致則重試
  • 寫端 API：WriteState(), WriteCommHeartbeat(), WriteMainHeartbeat()
  • 讀端 API：TryReadState()（最多重試 5 次），ReadCommHeartbeatTicks(), IsCommHeartbeatTimeout()
  • 底層檔案路徑預設：%TEMP%/robot_comm_shm.dat
  • 記憶體大小：128 bytes 固定

5.3 Robot.CommService/Program.cs

獨立行程通訊服務主程式。

  • PipeServerLoop()：等待主程式連線 → 逐行讀取 JSON 指令 → 呼叫 DeltaDriver 對應方法 → 回傳結果
  • HandleRequest()：switch 分派 13 種 IAxisCard 指令
  • StatusPushLoop()：每 10ms 將 DeltaDriver 狀態寫入共享記憶體 + 更新 CommService 心跳
  • HeartbeatMonitorLoop()：每 500ms 檢查主程式心跳，逾時 1s 觸發 SafeShutdown
  • SafeShutdown()：Estop → End → Dispose → 終止行程
  • 命令列參數：--mock, --zero-config=path, --shm-path=path

5.4 Robot.Driver.Delta/PipeAxisCard.cs

IAxisCard 遠端代理實作。

  • Start()：啟動 CommService 子行程 → 連接命名管道 → 開啟共享記憶體 → 啟動心跳計時器 → 發送 Start 指令
  • 屬性（Pos, Speed, State 等）：從共享記憶體讀取（零延遲），非走管道
  • 指令（End, MoveAbsolute 等）：透過管道同步發送請求 + 等待回應
  • 崩潰偵測：心跳逾時 / 行程終止 / 管道讀取失敗 → _commServiceDead=true → 回報 ALARM
  • IsCommServiceDead：公開屬性供上層查詢
  • Dispose()：釋放本端資源，不主動 kill CommService（由管道斷線觸發安全關機）

5.5 Robot.Driver.Delta/AxisCardFactory.cs

工廠方法：根據 useOutOfProcess 參數建立適當的 IAxisCard 實例。

// 生產環境（預設）
IAxisCard card = AxisCardFactory.Create(logger, useMock: false, useOutOfProcess: true);
// → 建立 PipeAxisCard，啟動 CommService 獨立行程

// 除錯 / 單行程模式
IAxisCard card = AxisCardFactory.Create(logger, useMock: true, useOutOfProcess: false);
// → 建立 DeltaDriver，在本行程內直接驅動
  • FindCommServicePath()：自動在同目錄、上層目錄尋找 CommService 執行檔
  • 可透過 commServicePath 參數手動指定路徑

6. 使用方式

6.1 上層整合範例

// 在 RA605RobotApp 或其他上層應用中：

var logger = new RobotLogger("MainApp");

// 建立軸卡（預設使用獨立行程模式）
IAxisCard axisCard = AxisCardFactory.Create(
    logger,
    zeroConfigPath: "axis_zero_config.json",
    useMock: false,
    useOutOfProcess: true   // 生產環境建議 true
);

// 使用方式與原本完全相同
axisCard.Start();
axisCard.Initial();
axisCard.MoveAbsolute(0, 90000, 0, 30000, 0, 0.5, 0.5);

// 偵測 CommService 崩潰
if (axisCard is PipeAxisCard pipe && pipe.IsCommServiceDead)
{
    logger.Fatal("CommService 已崩潰，進行 in-process 降級接管...");
    pipe.Dispose();

    // 降級為直連模式
    axisCard = new DeltaDriver(logger, useMock: false);
    axisCard.Start();
    axisCard.Estop(); // 確保安全
}
